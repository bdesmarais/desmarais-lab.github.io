Code Style

To make our code readable, which in turn facilitates the finding of bugs and eases software maintenance, we define acceptable code styles, that is, the manner in which code is written beyond that enforced by the language itself.

We most commonly use R, Python, and C++.

For R we specify adherance to the mlr group's style guide.

For Python we use PEP8.

For C++ we use Google's style guide.
